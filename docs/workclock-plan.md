# שעון עבודה (`workclock`) — תוכנית מסגרת

מסמך תכנון, יולי 2026. כלי-יום חדש במסגרת "כלי היום" של smrtTask
(`docs/day-tools-plan.md`). מוסיף **שכבת זמן וביצוע** מעל הדֶסק: טקס בוקר
מודרך → שעון-על שרץ כל היום → מצב ריצה עם הגבלות זמן מסלימות → סגירת יום
→ תיעוד ולמידה של זמן העבודה.

סטטוס: טיוטה לאישור המשתמש. מוקאפ הבאנר אושר (ראו §4).

---

## 1. העיקרון

- **כלי אופציונלי, ברירת מחדל כבוי** (כמו `planfocus`). מי שלא מדליק —
  לא רואה כלום. נשען על מסגרת כלי-היום הקיימת: רשומה אחת ברג'יסטרי, בלי
  מיגרציה למתג.
- **הכול קונפיגורבילי.** כל זמן, כל סף, כל שעת-סגירה — מגיע מקונפיג הכלי
  וניתן לשינוי במסך ההגדרות (§3).
- **הבאנר הוא chrome זמני של פעולה פעילה**, לא סרגל קבוע. הוא מופיע רק
  כששעון פעיל — בדיוק כמו שמסך המרתון הוא chrome של ריצה פעילה. זה
  מיישב את המתח עם עקרון ה-UI הקומפקטי (`day-tools-plan.md §1`).
- **מצב ריצה אמביינטי — לא מסך מלא.** החלטה (יולי 2026): המשתמש צריך
  גישה מלאה לאפליקציה תוך כדי ריצה (וואטסאפ, נכנס, מסמכים). לכן שעון
  המשימה הפעילה יושב **בבאנר**, והמשתמש עובד רגיל. מסך חוסם קופץ **רק**
  ברגע חריגה קשה ונסגר בתגובה — הוא לא "מצב הריצה" עצמו.

## 2. מכונת-המצבים של השעון

מצב אחד פעיל בכל רגע, נשמר ב-`useWorkClock` (§6.2):

| מצב | מתי | הבאנר מציג |
|---|---|---|
| `idle` | הכלי דלוק, שעון לא פעיל | הבאנר מוסתר |
| `offer` | פעילות ראשונה של היום במכשיר | הצעה: "להתחיל שעון עבודה?" |
| `ritual:inbox` | שלב 1 של הטקס | שעון-על + "בדוק נכנס" + ספירה-לאחור |
| `ritual:plan` | שלב 2 | "תכנן את היום — 3 בינוניות + 1 גדולה" + ספירה |
| `ritual:run` | שלב 3 | "התחל מצב ריצה" + ספירה |
| `run` | ריצה רגילה | שעון-על + שעון משימה פעילה + השהה/עצור |
| `paused` | השהיה | שעון קפוא + נדנוד "להמשיך?" |

- **מעברי הטקס** (`inbox → plan → run`): לחיצה על השורה נכנסת לשלב
  (מנווטת ליעד) ומשאירה אותו פעיל; בתום הטיימר של השלב — מעבר אוטומטי
  לשלב הבא. הזמנים מהקונפיג (`step_*_sec`).
- **`run` הוא מצב-על** שבתוכו מתקיימות רמות ההסלמה (§5) — אלו לא מצבים
  נפרדים אלא שכבות מעל `run`.

## 3. ההגדרות (קונפיג הכלי)

מפתחות **שטוחים** בתוך `day_tools.workclock` (שטוחים כי `resolveTool`
עושה מיזוג רדוד — `day-tools.ts:51`; אובייקט מקונן היה נדרס בשמירה חלקית).
כולם ניתנים לעריכה במסך "כלי היום" (§6.5), נחשפים רק כשהכלי דלוק.

| מפתח | ברירת מחדל | מה |
|---|---|---|
| `enabled` | `false` | מתג הכלי |
| `offer_daily` | `true` | להציע פתיחת שעון בכניסה הראשונה של היום |
| `step_inbox_sec` | `180` | טיימר שלב "בדוק נכנס" |
| `step_plan_sec` | `180` | טיימר שלב "תכנן יום" |
| `step_run_sec` | `30` | טיימר שלב "התחל ריצה" |
| `limit_quick_task_sec` | `300` | משימה מהירה → **אדום רך** (5 ד') |
| `limit_quick_total_sec` | `2700` | סה"כ מהיר → **באנר קופץ** (45 ד') |
| `limit_medium_task_sec` | `2700` | משימה בינונית → **מסך חוסם** (45 ד') |
| `limit_big_task_sec` | `10800` | משימה גדולה → **מסך חוסם** (3 ש') |
| `pause_nag_sec` | `300` | תדירות נדנוד ההשהיה (5 ד') |
| `sound_enabled` | `true` | צליל בנדנוד ההשהיה ובסגירת היום |
| `close_tz` | `"America/New_York"` | אזור-זמן לחישוב שעות הסגירה |
| `close_remind_at` | `"18:55"` | תזכורת "לסגור/להאריך" |
| `close_prompt_at` | `"19:20"` | חלון עצירה |
| `close_autostop_sec` | `300` | עצירה אוטומטית בלי מענה (5 ד') |

## 4. הבאנר — עיצוב (אושר)

מוקאפ אינטראקטיבי אושר. אנטומיה (RTL): מימין שעון-העל של היום
(`h:mm:ss`, tabular-nums) + אייקון; מפריד; במרכז תוכן תלוי-מצב (צ'יפ-שלב
+ טקסט + טיימר, או שם המשימה הפעילה + שעונה); משמאל פעולות (השהה/עצור).
שלושת המצבים הקופצים (הצעה, באנר קופץ, מסך חוסם) נפרשים מעל התוכן.

**החלטות עיצוב שנסגרו במוקאפ:**
- צ'יפ-שלב בשורה אחת (`nowrap`), לא תופס גובה.
- באנר קופץ (45 ד' מהיר): "הבנתי, ממשיך במהיר" (סוגר) · **"עבור
  לבינוניות"** = מסיים את סבב המהיר ומעביר את השעון הפעיל למשימה
  הבינונית/גדולה הראשונה שתוכננה להיום.
- מסך חוסם = רגע-החלטה בלבד: "סיימתי / עוד X דקות / פצל משימה". האפליקציה
  (וואטסאפ) נשארת מאחוריו.

## 5. מצב ריצה ורמות ההסלמה

בזמן `run` הבאנר מציג את **שעון המשימה הפעילה**. משימה נעשית "פעילה"
בלחיצת ▶ על שורת המשימה (§6.4) — בחירה מפורשת, לא אוטומטית בפתיחה, כדי
שלא יתחיל שעון בטעות. שעון המשימה הפעילה + סכומים מצטברים פר-גודל ליום
מוזנים ללוג (§7).

שלוש רמות, לפי הספים בקונפיג:

| רמה | טריגר (ברירת מחדל) | ביטוי |
|---|---|---|
| **אדום רך** | מהיר > 5 ד' | הפס והשורה מאדימים; חיווי בלבד, לא עוצר |
| **באנר קופץ** | סה"כ מהיר > 45 ד' | התראה נדבקת שדורשת סגירה; לא חוסם עבודה |
| **מסך חוסם** | בינוני > 45 ד' · גדול > 3 ש' | overlay מלא שנסגר רק בתגובה |

- **השהיה:** כפתור בבאנר בכל `run`. במצב `paused` — כל `pause_nag_sec`
  תזכורת ויזואלית + קולית (אם `sound_enabled`) להמשיך, כדי שלא יישכח
  ששעון עצור.
- **הצליל:** util חדש `src/lib/sound.ts` (Web Audio, כמו שתוכנן ל-
  `FocusSession` ב-`day-tools-plan.md §8.5` — למרתון אין צליל היום).

## 6. שינויי קוד (מיפוי מדויק)

### 6.1 רג'יסטרי + קונפיג — `src/lib/smrttask/day-tools.ts`
- הוספת `"workclock"` ל-`DayToolSlug`.
- רשומה ל-`DAY_TOOLS`: `{ slug: "workclock", defaultConfig: { ...§3 } }`.

### 6.2 סטייט לקוח — `src/hooks/useWorkClock.ts` (חדש)
- קאש ברמת-מודול + subscribers, כמו `useDayTools`/`useWorkCalendar`.
- מחזיק: `phase`, `startedAt` (חותמת, לא מונה — כדי שרענון יחשב נכון),
  `pausedSeconds`, `activeTaskId` + `activeTaskStartedAt`, סכומים פר-גודל,
  ומוני התראות.
- **התמדה:** `localStorage` (מיידי, שורד רענון) + heartbeat לשרת (§6.6).
  בטעינה: `GET /api/work-clock/today` מהמם את המצב.
- מריץ את הטיקים, הטיימרים של הטקס, בדיקת הספים וההסלמות, וחישוב שעות
  הסגירה מול `close_tz`.

### 6.3 הבאנר + השכבות — `src/components/smrttask/workclock/`
- `WorkClockBar.tsx` — הבאנר (מכונת-המצבים §2).
- `WorkClockOverlays.tsx` — הצעה / באנר קופץ / מסך חוסם / חלון סגירה.
- **הרכבה:** ב-`TabsWorkspace.tsx` בראש סביבת-העבודה — כך הבאנר שורד
  ניווט בין מסכים/פאנלים. מוצג רק כש-`workclock` דלוק ו-`phase !== idle`.

### 6.4 המשימה הפעילה — `TaskRow.tsx` + `TaskDetail.tsx`
- **אוטומטי (§9.5):** פתיחת חלון משימה במצב `run` מתחילה את שעונה
  וסוגרת את שעון הקודמת. סיום/סגירת המשימה עוצר את השעון.
- כפתור ▶ שקט על שורה נשאר כקיצור מפורש להפעלה בלי לפתוח את החלון.

### 6.5 הגדרות — `DayToolsSettings.tsx`
- הרחבה: מתחת לטוגל `workclock`, כשדלוק — שדות הקונפיג (§3). זה גם
  משלים את החוב מ-`day-tools-plan.md §2.3` ("קונפיג נחשף כשהכלי פעיל").

### 6.6 שרת — `server/src/modules/smrttask/workclock/routes.ts` (חדש)
כולם `requireAuth + requireOrg + requireApp("smrttask")`, דרך `api()`:
- `POST /api/work-clock/start` — פתיחה/חידוש של סשן היום.
- `PATCH /api/work-clock` — heartbeat: worked/paused/פר-גודל/מוני-התראות
  + span המשימה הפעילה. כל ~30–60ש' + על pause/stop.
- `POST /api/work-clock/stop` — סגירה עם `reason`.
- `GET /api/work-clock/today` — הסשן הנוכחי (rehydrate).
- `GET /api/work-clock/insights` — סטטיסטיקת למידה (שלב 4).

### 6.7 ניקוי (חלק מהשלבים)
- **מחיקת באנר "בוקר טוב"**: הסרת `MorningStartBanner` מ-`inbox/page.tsx`
  + מחיקת הרכיב ומפתחות `suggestions.morning*`. (בקשת המשתמש.)
- **הסרת מודל "בנה את היום" הכפול**: מחיקת `BuildDayBanner.tsx`, השורה
  העליונה, כפתור ה-☀️ וה-props הנלווים מ-`TaskList.tsx`. הבחירה נשארת
  ישירה (כפתור +/− על שורה). **שמירת `daily_plans` אוטומטית** בכל שינוי
  בחירה (קריאה ל-`commitDayPlan` מתוך `handlePlanToggle`, debounced) —
  כך הרצף/סטטיסטיקה נשמרים בלי כפתור "סיימתי".
- **מספור רשימת המהיר**: מספר קטן בצד כל שורה מהירה + צ'יפ ספירה על
  הכותרות (⚡ מהיר · N), לחיווי ויזואלי כמה מחכות.
- **הסרת `MorningInboxRedirect`**: מחיקת ההפניה האוטומטית לנכנס לגמרי
  מ-`TasksPageClient.tsx` + מחיקת הרכיב. (§9.2)
- **סדר סקשני הדֶסק**: ב-`TaskList.tsx` (method131 ON) — היום מהיר→גדול→
  בינוני; משנים ל-**מהיר→בינוני→גדול** (§9.6).

## 7. דאטה ולמידה

### 7.1 `work_sessions` (טבלה חדשה, שלב 1) + RLS
שורה אחת ליום, `UNIQUE (user_id, work_date)`. עמודות:
`started_at`, `ended_at`, `worked_seconds`, `paused_seconds`,
`quick_seconds`, `medium_seconds`, `big_seconds`,
`alerts_soft`, `alerts_popup`, `alerts_block`,
`ritual_completed bool`, `closed_reason text` — עם
`CHECK (closed_reason IN ('manual','auto','extended','open'))`.
כל שדה `*_seconds` עם `CHECK (>= 0)`. RLS אישי (`user_id = auth.uid()`),
כמו `daily_plans`/`focus_sessions`.

### 7.2 `work_task_spans` (טבלה חדשה, שלב 4) + RLS
פר-משימה: `session_id fk`, `task_id`, `size`, `seconds`, `started_at`,
`ended_at`. בסיס ללמידה גרנולרית (זמן ממוצע פר-גודל/פר-סוג).

### 7.3 עדכון rollover
כמו `daily_plans` (`20260712130000 pass 1`): לפני איפוס היום, סגירת כל
`work_sessions` פתוח (`closed_reason='auto'`) — רשת ביטחון אם המשתמש שכח
לעצור. שאר ה-rollover ללא שינוי.

### 7.4 למידה (שלב 4)
`GET /insights` + תצוגה קומפקטית: אורך יום ממוצע, זמן ממוצע פר משימה
מהירה/בינונית, מגמות, כמה חריגות. בהמשך — התאמת הספים המוצעים.

## 8. שלבי ביצוע

כל שלב עומד בפני עצמו, עובר את פרוטוקול ה-pre-push המלא (`CLAUDE.md`),
ומתמזג ל-`main` ב-`--no-ff` נפרד.

1. **מסגרת + שעון + לוג.** רג'יסטרי+קונפיג · שדות הגדרות · מיגרציית
   `work_sessions` + backend · `useWorkClock` · `WorkClockBar` בסיסי
   (שעון-על + התחל/עצור/השהה) · הצעת בוקר · התמדה+heartbeat · עדכון
   rollover. ניקוי: מחיקת באנר "בוקר טוב" + מספור רשימת המהיר.
   *(כבר נותן ערך: שעון-יום שרץ + תיעוד.)*
2. **טקס בוקר.** מכונת-המצבים של 3 השלבים + טיימרים + ניווט. הסרת מודל
   "בנה את היום" + שמירת `daily_plans` אוטומטית על בחירה.
3. **מצב ריצה + הסלמה.** ▶ על שורה (משימה פעילה) · שעון משימה בבאנר ·
   3 רמות ההסלמה · נדנוד השהיה (ויזואלי+קולי) · "עבור לבינוניות".
4. **סגירת יום + למידה.** טקס הסגירה (18:55/19:20/אוטומטי לפי `close_tz`)
   · `work_task_spans` · `/insights` + תצוגת תובנות.

## 9. החלטות שנסגרו (יולי 2026)

1. **טווח הבאנר** — הרכבה ב-`TabsWorkspace`; שורד ניווט בכל סביבת העבודה.
2. **`MorningInboxRedirect`** — **מוסר לגמרי.** ההפניה האוטומטית לנכנס
   נמחקת; טקס הבוקר של השעון הוא הכניסה היחידה לנכנס.
3. **`close_tz`** — ניו-יורק (ברירת מחדל, ניתן לשינוי).
4. **עצירה וחידוש באותו יום** — **אותה שורת-יום.** חידוש פעילות אחרי
   "עצור יום" ממשיך את אותו `work_sessions` (הזמן מצטבר). בחזרה אחרי
   עצירה — **מסך חידוש** מציע להמשיך את היום.
5. **בחירת משימה פעילה** — **אוטומטית.** פתיחת חלון משימה מתחילה את שעונה
   (וסוגרת את הקודמת). ▶ על השורה נשאר כקיצור מפורש.
6. **סדר סקשנים בדף המשימות** — ⚡מהיר → **בינוני → גדול** (גדול מתחת
   לבינוני). שינוי סדר הרינדור ב-`TaskList.tsx` (§6.7).
