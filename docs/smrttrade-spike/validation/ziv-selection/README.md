# ziv-selection — בדיקת-הסלקטיביות של זיו (מקורות + פרוטוקול)

בדיקה #1 מהתוכנית: האם בחירת-הכניסה של זיו מוסיפה תוחלת מעל המנוע ומעל אקראי,
דרך אותו מנוע-יציאה בדיוק, על אותם תאריכים? **תוצאות ומסקנות: `RESULTS.md`.**

## מבנה
| נתיב | תוכן |
|---|---|
| `ziv_fetch_manifest.json` | כל סרטוני-הזיו שנמשכו: `id, date, title, chars` |
| `transcripts/ziv_<date>_<id>.txt` | תמלול-כתוביות (iw) גולמי + כותרת-מטא. **קלט-החילוץ** — לאימות ידני מול הסרטון (`https://youtu.be/<id>`) |
| `verdicts/ziv_<date>_<id>.json` | פסקי-זיו שחולצו: `ticker, name, label, price_or_levels, quote, confidence`. **ציטוט-מגבה לכל פסק.** |
| `ziv_selection_test.py` | הבדיקה (ZIV מול ENGINE מול RANDOM) |
| `v1_market_entry_output.txt` | פלט-הריצה של v1 |

## מקורות
- **ערוץ:** זיו חקשוריאן — סייקלס טריידינג (`@cyclestrading`), סרטוני-ניתוח-מניות ציבוריים ב-YouTube.
- **משיכת-תמלול:** `yt-dlp --dump-json` לקבלת כתובת-הכתוביות (lang `iw`) + `curl` ישיר
  (ה-downloader של yt-dlp נחסם; המסלול הזה עובד). התמלולים הם כתוביות-אוטומטיות
  (שמות-חברות משוערים/משובשים → הטיקר מוסק בחילוץ).
- **פסקי-זיו ינו-פבר26 + 2023:** מקוריים ב-`../../goldenset/ziv_verdicts_*.json`.

## פרוטוקול אי-דליפה
1. **מחליט-הבדיקה = מנוע-יציאה דטרמיניסטי** (OHLCV בלבד).
2. **חילוץ-הפסקים = סוכנים נפרדים** שקראו **רק את התמלול** — לא מחירים, לא פלט-מנוע.
3. **נקודת-בזמן:** תאריך-ההחלטה = תאריך-הסרטון; כניסה בפתיחת-היום-הבא; אין הצצה קדימה.
4. **קבוצה בשלה בלבד:** תאריך-סרטון ≤ 2026-05-27 (≥~8 שבועות נתונים-קדימה נכון ל-2026-07).
   סרטונים טריים יותר נמשכו ל-`transcripts/` אך **מוחרגים** מהניתוח.

## הגדרות
- **"כניסת-זיו פעילה"** = `label ∈ {כניסה, מעקב·אזור-כניסה}`.
- **קונפאונד ידוע (ראה RESULTS §קונפאונד):** 59/62 מהפסקים הם `מעקב·אזור-כניסה`
  = "חכה-לפולבק", ולכן כניסה-בשוק (v1) אינה נאמנה לשיטה. v2 יתקן זאת.
