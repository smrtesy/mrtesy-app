# smrtPlan — מצב ומה שנשאר (מסמך העברה לסשן חדש)

מסמך הנדסי שמסכם את עיצוב "תכנון חופשי" של smrtPlan, מה כבר נבנה, ומה נשאר.
ענף עבודה: `claude/epic-hawking-xvp7bl` (מסונכרן עם `main`).

---

## ✅ מה כבר נבנה (הכל ב‑main + פרודקשן, מאומת)

1. **לוח עבודה** שני–שישי + סימון **חגי ישראל** (פסים אלכסוניים, נבדל מ"היום").
2. **סכמה** (הורצה על פרודקשן, פרויקט Smrtesy `exjnlghuzuvqedlltztz`):
   - `smrtplan_roles`, `smrtplan_role_members(is_primary)` (ראשי יחיד לתפקיד, partial‑unique).
   - `plans.status` (`draft`/`active`/`done`/`archived`, ברירת‑מחדל active), `plans.is_capability`, `plans.is_available`.
   - `role_id` על `tasks` / `smrtplan_stages` / `smrtplan_estimates`.
   - `smrtplan_dependencies.lag_days` (int).
3. **תפקידים ואיוש** — `RolesEditor`, אחראי ברירת‑מחדל = הראשי של התפקיד (בעת יצירת משימה).
4. **לייפסייקל** draft→active→done→archived; **משימות טיוטה שקטות** עד אישור
   (מסוננות מ‑`/plan/my-tasks`, `/plan/worker-tasks`, smrtTask `/tasks` + count, וה‑assistant).
5. **יכולות + מדף זמינות** — סימון כלי, השלמה→מדף (צ'יפ ירוק/אדום), מתג זמינות **מנותק** מ‑done/return.
6. **חיץ תלות (lag)** במנוע (backward+forward) + שדה "ימי חיץ" בכל תלות ב‑`PlanEffortDetail`.
7. **עורך משימה**: **ימים** במקום שעות (הוסר קטלוג/שעות/קיבולת מהעורך); **יצירה בצעד אחד**
   (אחראי+סטטוס+תאריך+משך).
8. **ליטוש לוח**: אות יום‑בשבוע (סגול חלש), קו שבוע עבה 3px, תיקון צ'יפ סטטוס (לא להציג על undefined).
9. **הסרת תפריטי הישן** מהלוח (קיבולת/שעות) — קבצי העורך וה‑API נשארו vestigial.
10. **מנוע**: תוכנית **effort** מתזמנת אחורה מ‑`end_date` שלה (משימות בלי due משלהן יורשות את יעד התוכנית).

## ⛔ מה שנשאר — לפי סדר בנייה

### #1 — מנוע: עוגן זרם + ירושת משך מהשלב (השלמת הליבה)
- פרק (episode) הוא העוגן ב‑stream: משימת‑תא יורשת את `due_date` של הפרק (לא של הזרם).
  דורש טעינת מבנה פרקים/תאים ל‑`computeOrgSchedule` (task → `episode_stage_status.task_id` → episode → `due_date`).
- **ירושת משך**: משך תא = דריסת‑התא ?? `default_duration_days` של השלב.
  צריך מיגרציה: עמודה `default_duration_days` על `smrtplan_stages`.
- **חיץ ביחס ליעד** (לא להתחלה): לשנות את אילוץ ה‑lag במנוע ל‑finish‑to‑finish.
- קבצים: `server/src/modules/smrtplan/engine.ts` (`dueOf`, backward/forward passes).

### #2 — בורר תלויות חוצה‑תוכניות
- בורר ה"תלויות" ב‑`PlanEffortDetail` מציג כרגע רק משימות מאותה תוכנית (`otherTasks`).
  להרחיב לבחירת משימה מתוכנית אחרת או **יכולת** (תלות מסוג `plan`).
- מנוע: לטפל בתלות מסוג `plan` (משימה תלויה ביכולת). היום נטענות רק `from_type=task,to_type=task`.

### #3 — חסימה‑מחדש + תג "מבוסס על" (תלוי ב‑#2)
- יכולת לא‑זמינה (`is_available=false`) → חוסמת מחדש תלויים **פתוחים** (לא משימות שהושלמו).
- יעד משימת‑התיקון **נגזר** = המוקדם מבין (מתי נדרש ראשון, מתי העובד המבצע יוצא — `constrains_user_id`).
- ויזואל: תג ירוק "✓ מבוסס על X" כשהספק יכולת שהושלמה; אדום "⛔ ממתין" כשלא‑זמין; חסום‑ללא‑תאריך מסומן דחוף.

### #4 — עורך הגרירה + inline (הכותרת הגדולה)
- גרירה/הצרה של ברים → שינוי תאריכים/משך (נעיצה ידנית; המנוע מכבד ומשלים את מה שלא נגעת בו).
- הזזת ציוני דרך; לחיצה על שורה → הוספת ציון דרך; מחיקה.
- **עריכת טקסט inline** (לחיצה על כותרת → עריכה → Enter שומר).
- "+ שורה" עם בחירת סוג (effort/stream/יכולת); drill‑down: משימות כברים עם **חיצי תלות**
  (ציור חץ = יצירת תלות + חיץ); כמה "ריבועים" בשורה אחת.
- "מצב תכנון חופשי" = toggle שמנהל ארגון מפעיל (יכולת ברמת org + מצב‑עריכה ברמת לוח/תוכנית).
- כלים מותקנים: `@dnd-kit/*`, `@xyflow/react`, `dagre`.
- זהירות: מיפוי פיקסל→עמודת‑יום‑עבודה חייב להשתמש במודל ה‑`colPos/xOf` של הלוח (הסתרת סופ"ש) + snap.
- recompute רץ org‑wide על כל PATCH → לעשות debounce/recompute‑on‑drop בגרירה.

### #5 — חצאי‑ימים + עומס
- `tasks.duration_days` int→numeric (מיגרציה) + קואורסיית `Number()` בקריאה במנוע → תמיכה ב‑0.5.
- **עומס** בתצוגת "ריכוז" (roster): סכום ימי‑המשימה של אדם בחלון מול ימי העבודה הזמינים → דגל עומס.
  בלי טבלת קיבולת — נגזר מהשיוכים + המשכים + לוח השנה.

### #6 — טמפלטים ("מרקטינג = 4 שלבים")
- טבלאות חדשות: תבנית = שלבים/משימות עם תפקידים + משכי ברירת‑מחדל + שרשרת תלות.
  "+ שורה → בחר מרקטינג" → מייצר משימות + תלויות + אחראים (לפי תפקיד).

## 🔒 החלטות עיצוב לשמר (לא לפתוח מחדש)

- שבוע עבודה **שני–שישי**, חגי ישראל (הצוות בניו‑יורק לפי חגי ישראל).
- משך ב**ימים/חצאי‑ימים** — **אין שעות, אין קיבולת פר‑אדם**. עומס = ימי‑משימה מול ימי‑עבודה.
- **תפקיד → אנשים** (ראשי = ברירת‑מחדל); שיוך לפי תפקיד, ניתן לדריסה ידנית (ידני גובר).
- **יעד יושב על התוצר**: effort = התוכנית עצמה; stream = כל פרק. **לא** פר‑משימה (דדליין פר‑משימה = חריג עם אילוץ חיצוני). תאריך התחלה = מחושב תמיד.
- **שני סוגי תאריכים**: יעד‑תוצר (עובדים אחורה) מול אילוץ‑תקרה (עובד עוזב / כלי לא זמין — מושך מוקדם; `הקצה = המוקדם מבין השניים`).
- **חיץ = פר‑משימה/חץ, ביחס לתאריך היעד** (finish‑to‑finish), בימי עבודה.
- **יכולת** (`is_capability`) = כלי חד‑פעמי לשימוש חוזר; הושלם→מדף; זמינות = מתג נפרד; לא‑זמין חוסם תלויים. נכס חי — אפשר לפתוח משימות תחזוקה.
- **זרם**: שלבים+משכים = ברירת‑מחדל ברמת הזרם; דריסה דלילה פר‑פרק; שינוי ברירת‑מחדל → כל הלא‑דרוסים מתעדכנים.
- מונח: בקוד `roster`; מול המשתמש **"ריכוז"**.
- עיקרון‑על: **פחות שלבים/חלונות** — מסך אחד לכל ההגדרות.

- i18n: כל מחרוזת דרך `useTranslations`/`getTranslations`, מפתח ב‑`src/messages/{he,en}.json` (שניהם באותו commit).

- מיגרציות: קבצים ממוספרים `supabase/migrations/YYYYMMDDHHMMSS_<slug>.sql`.

- push: feature branch → merge `origin/main` → fast‑forward `main` → push שניהם.
