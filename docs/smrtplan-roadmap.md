# smrtPlan — מצב ומה שנשאר (מסמך העברה לסשן חדש)

מסמך הנדסי שמסכם את עיצוב "תכנון חופשי" של smrtPlan, מה כבר נבנה, ומה נשאר.
ענף עבודה: `claude/epic-hawking-xvp7bl` (מסונכרן עם `main`).

---

## ✅ מה כבר נבנה (הכל ב‑main + פרודקשן, מאומת)

1. **לוח עבודה** שני–שישי + סימון **חגי ישראל** (פסים אלכסוניים, נבדל מ"היום").
2. **סכמה** (הורצה על פרודקשן, פרויקט Smrtesy `exjnlghuzuvqedlltztz`):
   - `smrtplan_roles`, `smrtplan_role_members(is_primary)` (ראשי יחיד לתפקיד, partial‑unique).
   - `plans.status` (`draft`/`active`/`done`/`archived`, ברירת‑מחדל active), `plans.is_capability`, `plans.is_available`.
   - `role_id` על `tasks` / `smrtplan_stages` / `smrtplan_estimates`.
   - `smrtplan_dependencies.lag_days` (int).
3. **תפקידים ואיוש** — `RolesEditor`, אחראי ברירת‑מחדל = הראשי של התפקיד (בעת יצירת משימה).
4. **לייפסייקל** draft→active→done→archived; **משימות טיוטה שקטות** עד אישור
   (מסוננות מ‑`/plan/my-tasks`, `/plan/worker-tasks`, smrtTask `/tasks` + count, וה‑assistant).
5. **יכולות + מדף זמינות** — סימון כלי, השלמה→מדף (צ'יפ ירוק/אדום), מתג זמינות **מנותק** מ‑done/return.
6. **חיץ תלות (lag)** במנוע (backward+forward) + שדה "ימי חיץ" בכל תלות ב‑`PlanEffortDetail`.
7. **עורך משימה**: **ימים** במקום שעות (הוסר קטלוג/שעות/קיבולת מהעורך); **יצירה בצעד אחד**
   (אחראי+סטטוס+תאריך+משך).
8. **ליטוש לוח**: אות יום‑בשבוע (סגול חלש), קו שבוע עבה 3px, תיקון צ'יפ סטטוס (לא להציג על undefined).
9. **הסרת תפריטי הישן** מהלוח (קיבולת/שעות) — קבצי העורך וה‑API נשארו vestigial.
10. **מנוע**: תוכנית **effort** מתזמנת אחורה מ‑`end_date` שלה (משימות בלי due משלהן יורשות את יעד התוכנית).
11. **עורך גרירה (#4)** — מצב עריכה בלוח (גישת full): גרירה/הצרה של ברי‑תוכנית לנעיצת
    התחלה/יעד, גרירה/הוספה/מחיקה של ציוני דרך, "+ שורה" (effort/stream/יכולת),
    עריכת כותרת inline. Drill‑down `PlanTaskGantt`: משימות כברים נגררים (נעיצת
    due+משך ידני), חיצי תלות ב‑SVG, ולחיצה‑לחיבור ליצירת תלות. מודל ציר משותף
    RTL‑safe (`src/lib/smrtplan/timeline.ts`, `useGanttDrag.ts`). המנוע/ראוטים ללא שינוי.

12. **#1 — מנוע: עוגן פרק + ירושת משך + חיץ finish-to-finish.** משימת‑תא מתזמנת
    מ‑`due_date` של הפרק; משך = דריסה ?? `default_duration_days` של השלב (עמודה חדשה,
    נקבעת במטריצה); החיץ נמדד מהיעד של התלוי עם רצפת‑ביטחון (lag=0 = ספק לפני התחלה).
13. **#2 — בורר תלויות חוצה‑תוכניות.** `GET /plans/:id/dep-candidates`; הבורר מקובץ
    (אותה תוכנית / תוכניות אחרות / יכולות). תלות ביכולת = `to_type=plan`; המנוע חוסם את
    תאריך ההתחלה של התלוי עד מסירת היכולת (`end_date`, מעבר קדימה).
14. **#3 — חסימה‑מחדש + תג "מבוסס על".** `attachNeedsHandoff` פותר תלויות‑יכולת:
    satisfied = done&&available; יכולת done‑אך‑לא‑זמינה חוסמת מחדש (`zoneOf`). תג ירוק
    "✓ מבוסס על" / אדום "⛔ ממתין". (משימת‑תיקון נגזרת — לא מומש; דורש יצירת משימות אוטומטית.)
15. **#5 — חצאי‑ימים + עומס.** `tasks.duration_days` → numeric (+`Number()` במנוע, קלט step 0.5);
    תג עומס ב"ריכוז" — ימי‑משימה פתוחים מול ימי עבודה זמינים (פחות סופ"ש/חגים).
16. **#6 — טמפלטים.** טבלאות `smrtplan_templates/_items/_deps`; עורך `TemplatesEditor`;
    "+ שורה → מתבנית" / כפתור "תבניות" → `apply` יוצר תוכנית עם משימות+אחראים+תלויות.

> **כל פריטי הרודמפ (#1–#6) הושלמו, נבדקו (build+tsc+תת‑סוכן), ונדחפו ל‑`claude/relaxed-ride-8kmba0`.
> שלוש המיגרציות הורצו על הפרודקשן (`exjnlghuzuvqedlltztz`).** ראה "ליטושים שנותרו" למטה.

## ✨ ליטושים אופציונליים שנותרו (לא חוסמים)

- **#3 — משימת‑תיקון נגזרת**: כשיכולת הופכת לא‑זמינה, האוטומציה שיוצרת *משימת תיקון* עם
  יעד נגזר (המוקדם מבין מתי נדרש / מתי העובד יוצא) לא מומשה — דורש החלטה מתי נוצרת המשימה.
  כרגע: התלויים פשוט נחסמים‑מחדש ויזואלית (תג אדום).
- **#4 — גרירת‑חץ אמיתית**: יצירת תלות ב‑Gantt היא לחיצה‑לחיבור; אפשר לשדרג ל‑drag עם xyflow.
- **#4 — "מצב תכנון חופשי" persistent**: מצב‑העריכה הוא state מקומי; אם רוצים דגל org ב‑DB.
- **#6 — duplicate dep**: הפרת `UNIQUE(from,to)` מחזירה 500 (ה‑UI מונע כפילות; אפשר 409).
- **חצאי‑ימים בתאריכי המנוע**: 0.5 נספר בעומס, אבל חישוב התאריכים מעגל ליום עבודה שלם.

## 🔒 החלטות עיצוב לשמר (לא לפתוח מחדש)

- שבוע עבודה **שני–שישי**, חגי ישראל (הצוות בניו‑יורק לפי חגי ישראל).
- משך ב**ימים/חצאי‑ימים** — **אין שעות, אין קיבולת פר‑אדם**. עומס = ימי‑משימה מול ימי‑עבודה.
- **תפקיד → אנשים** (ראשי = ברירת‑מחדל); שיוך לפי תפקיד, ניתן לדריסה ידנית (ידני גובר).
- **יעד יושב על התוצר**: effort = התוכנית עצמה; stream = כל פרק. **לא** פר‑משימה (דדליין פר‑משימה = חריג עם אילוץ חיצוני). תאריך התחלה = מחושב תמיד.
- **שני סוגי תאריכים**: יעד‑תוצר (עובדים אחורה) מול אילוץ‑תקרה (עובד עוזב / כלי לא זמין — מושך מוקדם; `הקצה = המוקדם מבין השניים`).
- **חיץ = פר‑משימה/חץ, ביחס לתאריך היעד** (finish‑to‑finish), בימי עבודה.
- **יכולת** (`is_capability`) = כלי חד‑פעמי לשימוש חוזר; הושלם→מדף; זמינות = מתג נפרד; לא‑זמין חוסם תלויים. נכס חי — אפשר לפתוח משימות תחזוקה.
- **זרם**: שלבים+משכים = ברירת‑מחדל ברמת הזרם; דריסה דלילה פר‑פרק; שינוי ברירת‑מחדל → כל הלא‑דרוסים מתעדכנים.
- מונח: בקוד `roster`; מול המשתמש **"ריכוז"**.
- עיקרון‑על: **פחות שלבים/חלונות** — מסך אחד לכל ההגדרות.
