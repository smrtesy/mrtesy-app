# ספק: איחוד הצעות / משימות / פלאן — "שולחן העבודה"

מסמך החלטות סגור, יוני 2026. סוכם בשיחת תכנון עם המשתמש — זה המקור
האחיד לבנייה. כל סטייה ממנו דורשת אישור מחדש.

## 1. מבנה-על

- **/inbox (נכנס) = החלטות בלבד.** הצעות AI + שיבוצי פלאן. שואף לאפס
  בכל בוקר (טקס: לסרוק → לאשר/לדחות הכל → לעבור למשימות).
- **/tasks = עמוד העבודה היחיד**, שלוש שכבות:
  - **על השולחן** — שני טורים: ⚡מהיר | רגיל. שורות קומפקטיות, גרירה
    לסידור. ללא תקרת גודל (בשלב זה).
  - **ממתינות** — כל המאושר שלא עכשיו, ממוין לפי דחיפות תאריך
    (אפקטיבי). בלי תאריך → בתחתית. זו ברירת המחדל הרגועה ל"אחר כך".
  - **הושלמו** — מתקפל, עם אפשרות החזרה (reopen).
  - חיפוש על כלל המשימות בראש הדף.
- **עלייה אוטומטית לשולחן:** יעד ≤ 3 ימי עסקים **וגם** לא חסומה.
  כלל תצוגה בלבד — לא שינוי דאטה (הזזת תאריך מורידה חזרה לבד).
- **ימי עסקים:** שני–שישי, בניכוי חגי ישראל — **אותה הגדרה כמו מנוע
  הפלאן** (`engine.ts isWorkingDay` + `smrtplan_blocked_days`). כל
  ספירת ימים במערכת (כלל 3 הימים, צבעי דחיפות, תוויות ותק) עוברת
  דרך util משותף שנחשף מהמנוע.
- /plan/my → הפניה ל-/tasks. עמודי תכנון הפלאן (גאנט, מטריצה, צוות,
  מאגר) נשארים ככלי תכנון.
- תצוגת היומן ב-/tasks (TaskCalendarView + טוגל) ועמוד /calendar — מוסרים.

## 2. שורת משימה (שולחן + ממתינות)

```
☐  כותרת  [✨/📋]  🏠  ⚡/רגיל        יושב 12 ימים   מחר 🔴   (hover: ⏰ ⬆/⬇)
```

- **✓ אחיד בכל המערכת** עם ביטול (toast) והחזרה מ"הושלמו". כפתור
  "השלם" הירוק הישן יורד. השלמה = status archived + completed_at
  (כמו היום), כולל לידת מופע חוזר 🔁.
- **תווית יעד צבועה** לפי דחיפות בימי עסקים: 🔴 עבר/היום, 🟠 ≤3,
  🟢 רחוק, ⚪ בלי תאריך. הדדליין המוצג = **אפקטיבי** (המוקדם מבין
  due_date ו-latest_finish של מנוע הפלאן) — אחיד בכל המערכת.
- **לחיצה על תווית היעד** פותחת בוחר תאריך לשינוי; תווית אפורה
  "בלי תאריך" — לחיצה מוסיפה יעד. **במשימות פלאן אין שינוי יעד**
  (מנהל בלבד, דרך עמודי התכנון) — התווית לא לחיצה.
- **תווית ותק:** מ-10 ימי עסקים ללא אינטראקציה (last_interaction_at) —
  תווית אפורה שקטה "יושב X ימים". מ-20 — נכללת בביקורת התקופתית.
- **"חזר מנדנוד":** צ'יפ ⏰ על פריט שהתעורר, נעלם באינטראקציה ראשונה
  (שדה woke_from_snooze_at). נדנוד חוזר **לשכבה שממנה בא**: הצעה
  לא-מאושרת → נכנס; משימה מאושרת → רשימות (לפי כלל 3 הימים).
  **אסור לנדנד מעבר ליעד** — חסימה/אזהרה בבוחר.
- שורת פלאן: זהה + 📋; חסומה: ⏳ במקום ✓ + שורת "ממתין ל: X";
  אין מחיקה (יש "בקש הסרה" — deletion_requested); תג ⚠קריטי.

## 3. כרטיס הצעה (נכנס)

- נשאר **כרטיס** (בהחלטה צריך הקשר), לא שורה.
- **בלי שורת תוויות.** מספר סידורי, מקור, סיווג, נימוק — הכל בפאנל ✨
  בלבד. על הכרטיס רק: כותרת + ✨, ⚡גודל (לחיץ), תווית יעד צבועה, תיאור.
- פעולות: ✏️ עריכה · ⏰ נודניק · 📝 שמור-כמידע │ ✗ דחה · ✗! דחה+סיבה ·
  ✔ אשר · ✓ בוצע-מיד. בחירה מרובה: אישור/דחייה/מיזוג גורפים (קיים).
- **✗! = "דחה ולמד":** דיאלוג הדחייה שומר את הסיבה גם כ-correction
  (`/api/corrections`, אותו מנגנון של הלוג) עם בחירת טווח:
  אישי / כלל כללי. הלוג נשאר ככלי ניתוח וייצוא.
- **שיבוץ פלאן בנכנס:** סוג הצעה חדש (assignment_status=proposed
  ומשויך אליי): כרטיס 📋 "שובצת למשימה…" עם קבל/דחה (accepted/declined).
- אישור הצעה: יעד ≤3 ימי עסקים → שולחן; אחרת → ממתינות.

## 4. כפתור הקשר — אחד לכל משימה, לפי מוצא

| מוצא | כפתור | תוכן הפאנל |
|---|---|---|
| AI | ✨ | מספר סידורי, מקור (שולח/נושא/מתי + פתח מייל), סיווג+ביטחון, גודל מוצע, נימוק, [👎 דחה ולמד] |
| פלאן | 📋 | תוכנית/שלב, מי שיבץ ומתי, משך משוער, **המקום בשרשרת** (קודם ✅ / אתה ▶️ / אחריך ⏳), ⚠קריטי, להתחיל-עד/לסיים-עד, [פתח את לוח התכנון ↗] |
| ידנית | — | אין (אין מספר סידורי ואין מקור) |

## 5. דיאלוג "חדש" — שני טאבים

- **טאב משימה (ברירת מחדל):** כותרת + Enter = נשמר. גודל ברירת מחדל
  **⚡מהיר** (טאגל לרגיל) → נכנסת ישר לטור המתאים על השולחן. שדות
  אופציונליים: 📍בית, 📅יעד (יעד רחוק → יורדת לממתינות עם הודעה),
  🔁חזרה. "עוד" מתקפל: תיאור, חומרים, שיוך לאדם. **בלי בורר פרויקט.**
- **טאב מידע:** כותרת (אופציונלי), טקסט, 📎 קובץ, בחירת פרויקט/תת-פרויקט
  עם **"+ חדש" במקום**. זוכר את הפרויקט האחרון שנבחר.

## 6. מרתון מהיר

- "▶ התחל ריצה" על הטור המהיר → מסך מלא, משימה אחת בכל פעם:
  **בוצע ✓ / דלג / זה-לא-מהיר** (מעביר לרגיל). טיימר + מונה "3/8".
- מסך סיום: "סיימת X משימות ב-Y דקות ⚡ — השיא: Z. קצב שיא: משימה
  כל M:SS". שובר שיא → קונפטי.
- טבלת `marathon_runs` (user_id, org_id, started_at, ended_at,
  completed_count, skipped_count) — שיאים והיסטוריה נגזרים ממנה.

## 7. AI

- **size בפרומפט ה-task builder** (אותה קריאה, שדה JSON נוסף):
  מהיר = פעולה אחת תחומה בלי הכנה (לענות/לאשר/להתקשר/לקבוע/לשלוח);
  רגיל = יצירה/הכנה/רב-שלבי/תלות באחרים. **בספק → רגיל** (שומר על
  אמינות רשימת המהיר למרתון).
- **context (בית) — ידני בלבד** בשלב זה. בלי ניחוש AI.
- תיקוני המשתמש (⚡↔רגיל, דחה-ולמד) נאספים; בהמשך יוזנו כחוקים
  ל-rules_memory אם יתגלו דפוסים.

## 8. פלאן — תוספות

- **`smrtplan_plans.manager_user_id`** — מנהל פר תוכנית (ריבוע בלוח
  התכנון, לא פר משימה).
- **התראת מנהל:** משימה בתוכנית שלו שנמצאת ≤3 ימי עסקים מהדדליין
  האפקטיבי **ועדיין חסומה** → התראה למנהל (notification קיים במערכת).

## 9. מנגנוני ריקון (נגד הצטברות)

- נכנס → מתרוקן בטקס בוקר (דחיות מהירות + גורפות + דחה-ולמד).
- ממתינות → **ביקורת תקופתית:** ≥20 ימי עסקים בלי מגע → באנר מרוכז
  "X משימות ישנות — עדיין רלוונטי?" עם שמור / נודניק / ארכב (גם גורף).
- תווית ותק מ-10 ימים (סעיף 2) — סימן מוקדם.
- אין תקרת שולחן בשלב זה (יישקל בעתיד אם יוצף).

## 10. מרכז מידע

- **לוח ריבועים** כרונולוגי (חדש למעלה) לכל פרויקט/תת-פרויקט. ריבוע:
  כותרת אופציונלית, טקסט, קישורים לחיצים, 📎 קבצים, חותמת תאריך+שעה,
  עריכה/מחיקה.
- **מרקדאון קל + קיצורי מקלדת:** Ctrl+B מודגש, Ctrl+I נטוי, Ctrl+K
  קישור; קישורים הופכים לחיצים אוטומטית; הדבקת קובץ/תמונה מצרפת.
  בלי עורך WYSIWYG מלא.
- **סיכום AI נעוץ למעלה:** כפתור בנה/עדכן; סיכום **אחד שנדרס**
  (הקודם נשמר ברקע); עריכה ידנית; "עודכן ב…"; חיווי התיישנות כשנוספו
  ריבועים אחרי הסיכום. קישורים בסיכום נשמרים **כלשונם** (עיקרון מערכת).
- **חיפוש** בריבועי הדף (כותרת+טקסט).
- **דף פרויקט-אב מציג גם את תתי-הפרויקטים**, עם צ'יפ סינון פר תת;
  דף תת-פרויקט נשאר זמין לפוקוס.
- **"הוסף כמידע"** מהצעה/משימה — מוסיף, לא מחליף; + יצירת פרויקט
  במקום; + תיקון הבאג כשאין פרויקט משויך (הטופס נתקע בלי יעד).
- **פרויקטים = מידע בלבד.** בורר הפרויקט יורד מיצירת/עריכת משימות;
  לשונית "הצעות פרויקטים" בנכנס תצומצם/תכובה בהמשך.

## 11. שינויי דאטה (מיגרציה אחת)

- `tasks.size` text CHECK ('quick','regular') NOT NULL DEFAULT 'regular'
- `tasks.context` text CHECK ('home','work') NULL
- `tasks.woke_from_snooze_at` timestamptz NULL
- `marathon_runs` (טבלה חדשה + RLS)
- `smrtplan_plans.manager_user_id` uuid FK → auth.users
- **בלי שינוי בסטטוסים** (inbox+manually_verified נשאר; עטיפת API
  בלבד אם יידרש בהמשך).
